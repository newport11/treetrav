Action Functionality Documentation


Rename

Rename allows a user to rename a folder. To perform this action:

  1. Type the origin folder path: Specify the current path of the folder you want to rename.
  2. Type the new folder name: Enter the new name you want to assign to the folder.
  3. The origin folder path must exist and be valid for the renaming to work.

Copy

Copy allows users to duplicate the contents of one folder into another location. To use this action:

  1. Type the origin folder path: Specify the path of the folder whose contents you want to duplicate.
  2. Type the destination folder path: Enter the path where you want to copy the contents to.
  3. Both the origin and destination folder paths must exist and be valid for the copy operation to work.

Move

Move is similar to copy, but instead of duplicating the contents of the folder, it moves the entire folder to the destination path. To execute this action:

  1. Type the origin folder path: Specify the path of the folder you want to move.
  2. Type the destination folder path: Enter the path where you want to move the folder to.
  3. Both the origin and destination folder paths must exist and be valid for the move operation to be successful. The folder is relocated to the new location, and it is no longer present in its original location.

These actions provide essential functionality for managing folders, allowing users to rename, copy, and move them as needed while ensuring the paths are valid for a successful operation.
